guide.js: Marker placement

Inline placement (best behaviour but restrictive)

Inline placement attaches the marker elements inside the target, ie: using $target.append(). The marker will move around with the target node, if its position changes.

This is best used when:

  1. the target is relative positioned
  2. the target can safely contain children nodes
  3. the target might move around

This placement mode uses absolute positioning and does not require any JavaScript manipulation to achieve the correct offset.

Sibling placement (less restrictive, slower)

Sibling placement attaches the marker elements right before or after the target, ie: using $target.before() or $target.after().

This is best used when:

  1. the target's container (first parent) is relative positioned
  2. the elements position changes in responsive design

This placement mode uses absolute positioning and manipulates margin properties to achieve the correct offset.

Overlay placement (most flexible, and slowest)

Overlay placement attaches the marker elements in an internal node generated by guide.js, which is appended to the body.

This is best used when:

  1. the position CSS property of the target, and its parent, can not be absolute, or relative
  2. design isn't responsive, or all devices support fixed positioning

The marker uses fixed positioning in overlay placement.
